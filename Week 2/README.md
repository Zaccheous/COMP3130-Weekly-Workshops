# COMP3130 – Mobile Application Development
## Week 2 Workshop – Wireframes and Mock-ups

## Project Overview
This workshop focuses on understanding the fundamentals of **mobile application UI/UX design**, specifically the creation of **wireframes and mock-ups**.

The goal of this task is to practice visual planning of a mobile application interface using modern design tools such as **Figma**.

The workshop consists of two main parts:

1. Creating **low-fidelity wireframes**
2. Creating **high-fidelity mock-ups with prototypes**



## Question 1 – Wireframes

Wireframes represent the **basic layout and structure** of an application screen.  
They focus on the placement of UI elements without detailed styling.

For this task:

- Wireframes were created using **Figma**
- The designs illustrate the **layout, navigation flow, and component structure**
- UI elements include:
  - Buttons
  - Icons
  - Navigation components
  - Content placeholders

### Steps Completed
1. Designed wireframes in **Figma**
2. Exported the wireframe screens as images
3. Uploaded the files to this repository
4. Committed the files using **Git**

### Files Included
- Wireframe design exports
- Source design files (if applicable)



## Question 2 – Mock-ups

Mock-ups are **high-fidelity visual designs** that represent how the final application interface will look.

Unlike wireframes, mock-ups include:

- Colours
- Typography
- Icons
- Buttons
- UI styling

### Tool Used
- **Figma**

### Task Requirements
- Create high-fidelity mock-ups of the application screens
- Apply visual design elements such as:
  - Icons
  - Styled buttons
  - Layout spacing
  - Colour schemes
- Connect screens using **Figma’s Prototype feature** to simulate navigation between screens

### Steps Completed
1. Designed high-fidelity UI screens
2. Added icons and styled buttons
3. Created interactive navigation using **Figma Prototype**
4. Exported the design files
5. Committed the files to Git
